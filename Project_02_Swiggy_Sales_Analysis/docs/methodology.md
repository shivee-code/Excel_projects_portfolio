# Methodology — Swiggy Sales & Customer Behavior Analysis

This project follows a structured Excel-based data analysis approach to ensure accuracy, consistency, and business relevance.

---

## 1. Data Understanding
- Reviewed raw order-level data containing sales, order dates, ratings, food items, location, and customer feedback
- Identified key fields required for KPI calculation and trend analysis

---

## 2. Data Cleaning & Transformation
- Handled missing and inconsistent values
- Standardized date formats and numeric fields
- Performed data validation to ensure accuracy

---

## 3. Feature Engineering
Created additional analytical columns to enable deeper insights:
- **Day:** Extracted weekday from order date
- **Week Number:** Identified weekly sales patterns
- **Quarter:** Grouped data into Q1–Q4 for performance comparison
- **Food Type (Veg / Non-Veg):**  
  Classified using keyword-based logic on food item names

---

## 4. KPI Calculation
Key performance indicators were calculated using Pivot Tables:
- Total Sales
- Total Orders
- Average Order Value (AOV)
- Average Rating
- Ratings Count

---

## 5. Analysis & Visualization
- Conducted time-based analysis (daily, weekly, monthly, quarterly)
- Compared Veg vs Non-Veg sales contribution
- Identified top-performing cities and states
- Built map-based geographic visualizations
- Designed interactive charts using slicers and filters

---

## 6. Dashboard Development
- Created a single-page, KPI-driven Excel dashboard
- Applied clean layout, consistent color themes, and icons
- Ensured usability for non-technical business users

---

## 7. Validation & Review
- Cross-checked KPI values with raw data
- Reviewed insights from a business stakeholder perspective
- Optimized dashboard performance and clarity

---

## Conclusion
This methodology ensures a reliable, repeatable, and business-focused Excel analytics workflow suitable for real-world decision-making.
